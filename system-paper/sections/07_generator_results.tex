\section{Generator Faithfulness}
\label{sec:generator}

% Isolates the generator: retrieval is replaced by gold (oracle) context, so
% any failure is the generator's. Source: mamai-eval faithfulness re-run
% configs/config-v0.2.0/reports/faithfulness-g4-vs-3n-prompts-20260622.html.
% This is where the deployment choice (Gemma 4 over 3n) is explained: the
% deciding axis is faithfulness, and small size is not the problem.

\subsection{Setup}
% Oracle = mamaretrieval chunks scored >= 5, top-3 (matched to deployed
% depth), n=2,989 questions. Failure = contradiction (wrong dose/threshold)
% or unsupported addition (a claim not in the source); incompleteness and
% refusals do NOT count. Evaluated across the matrix: Gemma 4 E4B vs
% Gemma 3n E4B (x baseline/G1/G1+G2), with Qwen3.5-397B as the frontier
% reference.

\subsection{Two-Pass Judge}
% Detector -> verifier. Patronus Lynx-70B flags every candidate faithfulness
% failure (high recall, low precision); GPT-5 re-reads each flag and keeps
% only genuine contradictions/unsupported additions. The "categorized
% true-hallucination" rate (deterministic, full population) is the metric of
% record. Judge-selection journey worth one line: MiniCheck (too small,
% unauditable) and Qwen-397B (circular -- it built the oracle labels) were
% rejected; Lynx is independent (Llama-3 family) and auditable.

\subsection{Results}
% Categorized true-hallucination, across baseline/G1/G1+G2:
%   Gemma 4 E4B:   2.64% / 3.31% / 3.55%
%   Gemma 3n E4B:  6.26% / 6.46% / 6.72%
%   Qwen3.5-397B:  2.51% / 3.71% / 0.97%
% Gemma 4 is ~2x more faithful than Gemma 3n at every prompt, and grounds
% about as faithfully as the unconstrained frontier Qwen. So the small
% on-device Gemma 4 is already strong on faithfulness; Gemma 3n is the
% outlier, not the small size. This is the core reason Gemma 4 ships.

\subsection{The Calibration Artifact (honesty note)}
% The headline metric is the categorized rate above. A separate Lynx-miss-
% rate "calibrated" extrapolation produced a 53.9% outlier on Gemma 4 +G1+G2
% -- a statistical artifact: G2's longer answers (1,193 vs 770 chars) exceed
% the narrow 3-chunk oracle, so standard-of-care content gets flagged as
% unsupported, and a ~50-sample miss-rate swings the population estimate.
% Report the deterministic categorized rate; explain the artifact (section
% 4a of the source report) rather than burying it.

\subsection{Implication}
% Generator matters ~4x more than prompt for faithfulness, and 3n's gap is a
% property of the model (it contradicts context ~2x as often), not the
% prompt. So improving faithfulness -- and thus enabling a more helpful model
% to be deployed safely -- requires generator-side work: RAG-grounding /
% fine-tuning (Section ref sec:discussion), not more prompt engineering.
