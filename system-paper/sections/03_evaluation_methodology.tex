\subsection{Bottom-Up, Presented Top-Down}
% The system is evaluated in layers -- end-to-end, then retriever and
% generator in isolation, then latency. Built bottom-up (a weak retriever
% would make generator eval meaningless; end-to-end alone confounds the
% two), but PRESENTED top-down: lead with the deployed-system result
% (Section ref sec:endtoend), then decompose to show where it comes from
% (Sections ref sec:retrieval-results, ref sec:generator).

\subsection{Benchmarks Used}
% mamaretrieval v0.2.0 (3,185 queries; 230,964 labeled query-chunk pairs
% from the Tier 3 top-20 x 6-retriever pool): retriever evaluation and
% oracle context for generator isolation.
% mamabench v0.2.1 (25,949 rows across 7 sources, 3 set_types: mcq /
% open_ended / open_ended_rubric): end-to-end evaluation. Construction
% details in the companion benchmarks paper [cite paper2].

\subsection{Device--Cluster Calibration}
% Most evaluation runs on cluster (llama-cpp Q4_0 on A100) rather than
% on-device (LiteRT-LM FP16 on Android GPU). Justified by a paired MCQ
% calibration on 300 rows: accuracy delta +2.7 pp (within noise),
% per-row agreement 65.0%, Cohen's kappa 0.558 -- the two runtimes are
% effectively interchangeable for aggregate scoring.
% Source: mamai-eval configs/config-v0.2.0/reports/calibration-mcq-20260519.md.

\subsection{Judge Validation}
% The open-ended tracks are scored by an LLM judge -- so the judge itself is
% validated first. Phase A: 5 candidate judges (Maverick, Nemotron,
% gpt-oss-120b, gpt-5-mini, gpt-5) run the HealthBench rubric over 6,853
% physician-labeled triples (mamabench v0.2.1 calibration side-file). No
% judge hits the physician baseline; larger open models over-rate
% (rubber-stamp), gpt-5 under-rates by ~19 pp. gpt-oss-120b is the best
% calibrated (met-rate -3.8 pp vs physicians; NOT-MET agreement 67.9%, no
% rubber-stamping) and is pinned for all open-ended scoring. Disclose the
% conservative bias as a known direction. Source:
% docs/judge-validation-phase-a-result-20260608.html.

\subsection{Metric Choices}
% End-to-end (open-ended, deployment-relevant): key-fact recall + 4-level
% safety verdict (Kenya); weighted_met decomposed into positive credit vs
% penalty (HealthBench rubric). MCQ accuracy as a negative control only.
% Retrieval: Hit Rate and precision at top-3 (deployed depth), lenient
% (>= 3) / strict (>= 5) on the 0--6 graded rubric, + weighted [cite paper2].
% Generator: categorized true-hallucination (two-pass Lynx-70B -> GPT-5).
% Latency: median per condition over repeated runs.
